## 1. 개요

### 제품명
**SignalTracker** — 구조화된 불확실성 관리 도구

### 한 줄 목표
실제 데이터 소스에서 투자 판단 신호를 추출하고, 예측 정확도를 누적 추적해 VC의 의사결정 인풋으로 쓰는 시스템.

### 배경

**해결하려는 문제:**
기존 MiroFish류 시스템의 근본 한계는 "LLM 한 명이 혼자 역할극"을 한다는 것이다. 정보 독립성이 없다. 예측력이 아니라 그럴듯함만 있다.

진짜 문제는 따로 있다. VC는 매일 수십 개의 뉴스, 보고서, 정책 문서를 본다. 이 정보들 사이의 패턴을 손으로 정리하는 데 시간이 너무 많이 든다. 그리고 내 판단이 실제로 얼마나 맞았는지 기록이 없다.

**타겟 사용자:**
1차 타겟: Ethan (CIO, TheVentures) — 한국 AI 스타트업 투자 타이밍 판단에 쓸 개인 도구  
2차 타겟: K-AI Fund 운영 시 팀 공용 도구로 확장 가능

**시장 기회:**
"AI 투자 인텔리전스" 도구는 많지만, 예측 정확도를 추적하고 calibration을 쌓아가는 도구는 없다. 맞고 틀린 걸 기록하는 것 자체가 차별점이다.

### 성공 지표

**정량 (3개월 기준):**
- 주 3회 이상 실제 사용
- 예측 로그 50개 이상 누적
- 예측 정확도 백테스트 가능한 상태

**정성:**
- "LP 미팅 전에 반드시 돌려보게 된다"는 사용 습관 형성
- 내 판단과 시스템 판단이 엇갈렸을 때 어느 쪽이 맞는지 확인 가능

## 4. 사용자 플로우

### 주요 플로우

```
[소스 입력] → [Claude가 신호 추출] → [앙상블 종합] → [예측 출력 + 저장] → [3개월 후 정확도 확인]
```

### 상세 플로우

**Step 1 — 입력**
- 뉴스 URL, 보고서 텍스트, 또는 간단한 메모를 입력
- 예측 주제를 한 줄로 정의 ("이 정책이 한국 AI SaaS 투자 타이밍에 유리한가")

**Step 2 — 신호 추출**
- Claude API가 각 소스에서 신호 추출
- 각 신호에 방향(bullish/bearish/neutral)과 강도(1-5) 태깅
- 소스가 여러 개일 경우 신호 간 모순 탐지

**Step 3 — 예측 종합**
- 신호들을 가중 평균으로 종합
- 최종 판단: bullish/bearish/neutral + 신뢰도(%)
- 핵심 근거 3개 자동 요약

**Step 4 — 저장**
- 예측 결과를 로컬 DB에 자동 저장
- 날짜, 주제, 예측값, 신뢰도, 소스 메타데이터 기록

**Step 5 — 결과 기록 (나중에)**
- 예측 후 실제 결과를 사용자가 직접 입력
- 정확도 누적 → calibration 그래프 생성

## 5. 기술 요구사항

### 전체 구조

```
[Claude Code CLI 환경]
  ├── 데이터 수집 모듈 (Python)
  │     ├── URL 파싱 / 텍스트 입력
  │     └── (추후) RSS 피드 자동 수집
  ├── 신호 추출 모듈 (Claude API)
  │     ├── 소스별 신호 추출
  │     └── 신호 간 모순 탐지
  ├── 예측 엔진 (Python)
  │     ├── 신호 앙상블
  │     └── 신뢰구간 계산
  ├── 저장 모듈 (SQLite)
  │     ├── 예측 로그
  │     └── 정확도 기록
  └── 대시보드 (간단한 웹 UI)
        ├── 예측 결과 확인
        └── calibration 그래프
```

### 기술 스택

**백엔드:**
- Python 3.11+ (Claude Code로 빌드)
- Claude API (claude-sonnet-4-20250514)
- SQLite (로컬 저장, 복잡한 DB 불필요)

**프론트엔드:**
- 초기: CLI 인터페이스 (Claude Code 터미널)
- v2: 간단한 React 웹 UI

**외부 연동:**
- Anthropic Claude API (필수)
- Newspaper3k / BeautifulSoup (URL 파싱)
- (추후) DART API (공시 데이터), 뉴스 RSS

### 성능 요구사항

- 신호 추출 응답: 30초 이내 (소스 1개 기준)
- 동시 사용자: 1명 (개인 도구)
- 저장 용량: 로컬 SQLite, 1GB 이하

## 6. UI/UX 요구사항

### 디자인 원칙

- **CLI 우선**: 복잡한 UI 없이 터미널에서 바로 작동
- **결과 가독성**: 예측 결과는 한눈에 읽히는 구조화된 출력
- **저장 투명성**: 무엇이 어디 저장됐는지 항상 확인 가능

### 핵심 화면 (웹 UI v2 기준)

1. **입력 화면**: URL/텍스트 입력 + 예측 주제 정의
2. **결과 화면**: 신호 목록 + 앙상블 결과 + 신뢰도
3. **로그 화면**: 과거 예측 목록 + 결과 입력 버튼
4. **대시보드**: calibration 그래프 + 주제별 정확도

## 부록: MiroFish와 결정적 차이

| | MiroFish | SignalTracker |
|---|---|---|
| 정보 소스 | LLM 상상 | 실제 문서/뉴스 |
| 예측 방식 | 역할극 앙상블 | 신호 추출 앙상블 |
| 정확도 추적 | 없음 | 핵심 기능 |
| Calibration | 없음 | 누적 추적 |
| 실용성 | 그럴듯함 | 판단 보조 도구 |

핵심 차이는 하나다. **예측 기록이 쌓인다.** 맞고 틀린 것을 추적하는 시스템만이 calibration을 만든다. Calibration이 없으면 예측력이 없다.
